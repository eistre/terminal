\begin{abstract}
Linux and its command line play an important role in courses at the University of Tartu, such as ``Operating Systems'' and ``Computer Security''.
To support hands-on practice in these courses, earlier thesis work created a browser-based Linux command-line learning environment and later extended it into a cloud-hosted platform for continuous course use.
These earlier iterations showed the educational value of browser-based Linux practice, but they also revealed shortcomings in maintainability, deployment reproducibility, portability across environments, and cost efficiency.
The aim of this thesis was to redesign the platform into a more sustainable and portable system while preserving its educational purpose and core workflow.

The work introduced a revised architecture with clearer modular boundaries, a provider-agnostic provisioning layer for user-specific Linux environments, and improved authentication and multilingual content management.
Automated environment setup was implemented using Helm and Terraform.
The redesigned platform was deployed and evaluated in the University of Tartu HPC Kubernetes cluster, Microsoft Azure, and Amazon Web Services.

The evaluation carried out during real course use in February and March 2026 showed that the revised platform preserved the intended educational workflow, and students were able to access the environment, start interactive Linux sessions, and complete the relevant tasks without notable operational incidents.
Compared with the previous Azure deployment, the revised Azure architecture significantly reduced the baseline cost of keeping the platform available.
Among the environments tested, the University of Tartu HPC deployment appeared to be the most suitable option for institutional use, while Azure was the strongest public cloud alternative.
\end{abstract}

\begin{otherInfo}{estonian}{Veebipõhise Linuxi käsurea õppekeskkonna kaasajastamine ja kulude optimeerimine}
\begin{abstract}
Linux ja selle käsurida mängivad olulist rolli Tartu Ülikooli kursustel ``Operatsioonisüsteemid'' ja ``Andmeturve''.
Nende kursuste praktiliste harjutuste toetamiseks loodi varasemate lõputööde raames veebipõhine Linuxi käsurea õppekeskkond, mida hiljem laiendati pilvepõhiseks platvormiks.
Käesoleva lõputöö eesmärk oli kujundada olemasolev platvorm ümber jätkusuutlikumaks ja paremini ülekantavaks süsteemiks, säilitades samal ajal selle haridusliku eesmärgi ja põhilise töövoo.

Töös võeti kasutusele uus arhitektuur, millel olid selgemad moodulipiirid, teenusepakkujast sõltumatu kasutajapõhiste Linuxi keskkondade haldamiskiht ning täiustatud autentimine ja mitmekeelse sisu haldus.
Keskkondade automatiseeritud ülesseadmiseks kasutati tehnoloogiaid Helm ja Terraform.
Uuendatud platvorm võeti kasutusele ja hinnati Tartu Ülikooli HPC Kubernetese klastris ning keskkondades Microsoft Azure ja Amazon Web Services.

Testimine, mis viidi läbi 2026. aasta veebruaris ja märtsis, näitas, et uuendatud platvorm säilitas esialgse haridusliku töövoo ning tudengid said keskkonda sisse logida, alustada interaktiivseid Linuxi sessioone ja lahendada ettenähtud ülesandeid ilma märkimisväärsete probleemideta.
Võrreldes varasema Azure'i keskkonnaga vähendas uuendatud Azure'i arhitektuur oluliselt platvormi pideva käigushoidmise baaskulu.
Testitud keskkondade hulgast näis Tartu Ülikooli HPC keskkond olevat pikemaajaliseks kasutamiseks sobivaim ning avalikest pilveteenustest paistis Azure tugevaima alternatiivina.
\end{abstract}
